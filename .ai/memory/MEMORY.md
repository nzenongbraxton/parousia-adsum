## Post-task discipline (established 2026-06-04)

After every full task completion:
1. Update `README.md` if the project state changed (new features, changed commands, updated architecture).
2. Update this file (`MEMORY.md`) with a log entry describing what changed and why.
3. Stage changes, but **always ask the user for approval first** after verifying that the changes work perfectly and to their taste, before committing. Do not commit automatically without explicit user confirmation.
